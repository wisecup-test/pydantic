<rule_activation id="69eb9398-b1c2-4bf0-8b5f-d17183723c1f" title="Adopt Environment Variable-Based Secret Management for External API Integration: Actual Credential Values" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files matching `**/*`. Environment variable-based secret management is mandatory for all external API integrations requiring authentication credentials.
</rule_activation>

### Rules

- **R-SEC-001** MUST_NOT: Actual credential values MUST NOT be committed to version control, and .env files containing real secrets MUST be excluded via .gitignore
- **R-SEC-002** MUST: Use os.environ.get() or os.getenv() in Python with appropriate default handling and validation for all API credentials
- **R-SEC-003** MUST: Create a .env.example file documenting all required environment variables with placeholder values (e.g., GITHUB_API_TOKEN=your_token_here)
- **R-SEC-004** MUST: Add .env to .gitignore to prevent accidental commits of actual credentials
- **R-SEC-005** MUST: Implement startup validation that checks for required environment variables and fails fast with clear error messages
- **R-SEC-006** MUST: Document environment variable requirements in README.md and deployment documentation
- **R-SEC-007** SHOULD: Use consistent naming conventions: SERVICE_NAME_API_KEY, SERVICE_NAME_TOKEN, SERVICE_NAME_SECRET
- **R-SEC-008** SHOULD: Implement logging filters to redact sensitive environment variables; use structured logging with secret masking
- **R-SEC-009** SHOULD: Use CI/CD platform secret management features (GitHub Secrets, GitLab CI/CD variables) and restrict access to secret configuration

### Verify

```bash
# Check for hardcoded API credentials in Python files
grep -r "api[_-]key\s*=\s*['\"]" --include="*.py" --exclude-dir=tests --exclude-dir=.venv | grep -v "os.environ" | grep -v "getenv"

# Check for hardcoded tokens in Python files
grep -r "token\s*=\s*['\"]" --include="*.py" --exclude-dir=tests --exclude-dir=.venv | grep -v "os.environ" | grep -v "getenv"

# Verify .env files are properly excluded from version control
find . -name ".env" -not -path "*/node_modules/*" -not -path "*/.venv/*" | xargs git check-ignore -v
```

**Accept when:**
- No hardcoded API credentials are found in source code (excluding test mocks)
- All .env files containing actual secrets are properly excluded from version control
- All external API integrations retrieve credentials from environment variables at runtime
- Documentation clearly specifies required environment variables for each integration
- .env.example file exists with placeholder values for all required environment variables
- Startup validation checks are implemented for required environment variables

<enforcement>
Claude Code MUST NOT skip or defer verification. All pull requests containing hardcoded credentials must be rejected. Exposed credentials must be revoked and regenerated before code can be merged. Security team must be notified of any credential exposure incidents.
</enforcement>
