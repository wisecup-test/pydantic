<rule_activation id="55f84847-73ab-4d06-b84c-24d998a0bdc6" title="Adopt Environment Variable-Based Secret Management for External API Integration: Example Configuration Files" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files matching `**/*`. Environment variable-based secret management is required for all external API integrations to prevent credential exposure in version control and maintain security best practices.
</rule_activation>

### Rules

- **R-SEC-001** SHOULD: Example configuration files (.env.example) SHOULD be provided with placeholder values for required API credentials.

### Verify

```bash
# Check for hardcoded API credentials in Python files
grep -r "api[_-]key\s*=\s*['\"]" --include="*.py" --exclude-dir=tests --exclude-dir=.venv | grep -v "os.environ" | grep -v "getenv"

# Check for hardcoded tokens in Python files
grep -r "token\s*=\s*['\"]" --include="*.py" --exclude-dir=tests --exclude-dir=.venv | grep -v "os.environ" | grep -v "getenv"

# Verify .env files are properly excluded from version control
find . -name ".env" -not -path "*/node_modules/*" -not -path "*/.venv/*" | xargs git check-ignore -v
```

**Accept when:**
- No hardcoded API credentials are found in source code (excluding test mocks)
- All .env files containing actual secrets are properly excluded from version control
- All external API integrations retrieve credentials from environment variables at runtime
- Documentation clearly specifies required environment variables for each integration
- A .env.example file exists with placeholder values for all required API credentials

<enforcement>
Claude Code MUST NOT skip or defer verification. All pull requests and code changes must pass the credential scanning checks before acceptance. Violations require immediate credential rotation and security team notification.
</enforcement>
