### Rules

- **R-PYDANTIC-001** MUST: Validator implementations MUST be tested against pydantic-core's validation engine to ensure compatibility.
- **R-PYDANTIC-002** MUST: All Pydantic V1 imports MUST use the pydantic.v1 namespace, not direct pydantic imports (e.g., `from pydantic.v1 import BaseModel`).
- **R-PYDANTIC-003** MUST: Type checking tests MUST be placed in tests/typechecking/ directory following existing patterns (type_adapter.py, json_schema_examples.py, pipeline_api.py).
- **R-PYDANTIC-004** MUST: Custom validators MUST be tested against pydantic-core validation engine as shown in pydantic-core/tests/validators/test_with_default.py.
- **R-PYDANTIC-005** SHOULD: Use Pydantic V2 TypeAdapter for dynamic validation: `TypeAdapter(MyType).validate_python(data)`.
- **R-PYDANTIC-006** SHOULD: Use model_config for Pydantic V2 configuration instead of Config class from V1.
- **R-PYDANTIC-007** SHOULD: Generate JSON schemas using model.model_json_schema() in V2 or model.schema() in V1 compatibility mode.
