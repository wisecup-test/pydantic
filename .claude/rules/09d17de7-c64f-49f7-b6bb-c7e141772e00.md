<rule_activation id="09d17de7-c64f-49f7-b6bb-c7e141772e00" title="Adopt Environment Variable-Based Secret Management for External API Integration: Applications Retrieve Credentials" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files matching `**/*`. Applications MUST retrieve API credentials from environment variables at runtime, never hardcoding secrets in source code.
</rule_activation>

### Rules

- **R-SEC-001** MUST: Applications MUST retrieve API credentials from environment variables at runtime using standard environment variable access patterns (os.environ.get(), os.getenv(), or equivalent).
- **R-SEC-002** MUST: All external API integrations requiring authentication (GitHub API, Algolia, third-party services) MUST use environment variables for API keys, tokens, and OAuth credentials.
- **R-SEC-003** MUST: A .env.example file MUST be created documenting all required environment variables with placeholder values (e.g., GITHUB_API_TOKEN=your_token_here).
- **R-SEC-004** MUST: .env files containing actual secrets MUST be added to .gitignore to prevent accidental commits.
- **R-SEC-005** MUST: Startup validation MUST check for required environment variables and fail fast with clear error messages when missing.
- **R-SEC-006** SHOULD: Use consistent naming conventions for environment variables: SERVICE_NAME_API_KEY, SERVICE_NAME_TOKEN, SERVICE_NAME_SECRET.
- **R-SEC-007** SHOULD: Environment variable requirements MUST be documented in README.md and deployment documentation.
- **R-SEC-008** MAY: Unit tests may use hardcoded mock credentials when testing against mock API servers with no external connectivity (EXC-001).

### Verify

```bash
# Scan for hardcoded API credentials in Python files
grep -r "api[_-]key\s*=\s*['\"]" --include="*.py" --exclude-dir=tests --exclude-dir=.venv | grep -v "os.environ" | grep -v "getenv"

# Scan for hardcoded tokens in Python files
grep -r "token\s*=\s*['\"]" --include="*.py" --exclude-dir=tests --exclude-dir=.venv | grep -v "os.environ" | grep -v "getenv"

# Verify .env files are properly excluded from version control
find . -name ".env" -not -path "*/node_modules/*" -not -path "*/.venv/*" | xargs git check-ignore -v
```

**Accept when:**
- No hardcoded API credentials are found in source code (excluding test mocks)
- All .env files containing actual secrets are properly excluded from version control
- All external API integrations retrieve credentials from environment variables at runtime
- Documentation clearly specifies required environment variables for each integration
- A .env.example file exists with placeholder values for all required environment variables
- Startup validation checks are implemented for required environment variables

<enforcement>
Claude Code MUST NOT skip or defer verification. All pull requests containing hardcoded credentials MUST be rejected. Exposed credentials MUST be revoked and regenerated before code can be merged. Security team MUST be notified of any credential exposure incidents.
</enforcement>
