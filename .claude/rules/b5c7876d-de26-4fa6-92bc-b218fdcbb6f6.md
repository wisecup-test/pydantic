<rule_activation id="b5c7876d-de26-4fa6-92bc-b218fdcbb6f6" title="Adopt Pydantic V1 Compatibility Layer for Type Validation: Json Schema Generation" applies_to="**/*">
These rules are ALWAYS ACTIVE for all Python modules requiring runtime type validation, data serialization, schema generation, API request/response validation, configuration parsing, and type checking in test suites.
</rule_activation>

### Rules

- **R-PYDANTIC-001** SHOULD: JSON schema generation SHOULD use Pydantic's built-in schema generation capabilities rather than custom implementations.
- **R-PYDANTIC-002** MUST: All Pydantic V1 imports MUST use the `pydantic.v1` namespace, not direct `pydantic` imports (e.g., `from pydantic.v1 import BaseModel`).
- **R-PYDANTIC-003** SHOULD: Use Pydantic V2 TypeAdapter for dynamic validation: `TypeAdapter(MyType).validate_python(data)`.
- **R-PYDANTIC-004** MUST: Place type checking tests in `tests/typechecking/` directory following existing patterns (type_adapter.py, json_schema_examples.py, pipeline_api.py).
- **R-PYDANTIC-005** SHOULD: Test custom validators against pydantic-core validation engine as shown in test patterns.
- **R-PYDANTIC-006** SHOULD: Use `model_config` for Pydantic V2 configuration instead of Config class from V1.
- **R-PYDANTIC-007** SHOULD: Generate JSON schemas using `model.model_json_schema()` in V2 or `model.schema()` in V1 compatibility mode.

### Verify

```bash
# Verify all Pydantic V1 imports are properly namespaced
grep -r 'from pydantic import' --include='*.py' . | grep -v 'from pydantic.v1 import' | grep -v 'from pydantic import' || echo 'V1 imports properly namespaced'

# Verify type checking tests exist
find . -path '*/tests/typechecking/*.py' -type f | wc -l | awk '{if($1>0) print "Type checking tests found"; else exit 1}'

# Verify Pydantic imports work without conflicts
python -c 'import pydantic; from pydantic.v1 import BaseModel; from pydantic import TypeAdapter; print("Pydantic imports successful")'
```

**Accept when:**
- All Pydantic V1 imports use the `pydantic.v1` namespace, not direct pydantic imports
- Type checking test files exist in `tests/typechecking/` directory covering TypeAdapter, JSON schema, and pipeline APIs
- Pydantic and `pydantic.v1` modules can be imported successfully without conflicts
- Validator tests demonstrate compatibility with pydantic-core validation engine
- JSON schema generation uses Pydantic's built-in capabilities, not custom implementations
- All data models include validation logic
