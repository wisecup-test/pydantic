These rules are ALWAYS ACTIVE for all Python test files in the tests/ directory.
</rule_activation>

### Rules

- **R-PYTEST-001** MUST: Test files MUST be organized in a dedicated tests/ directory structure that mirrors the source code organization.
- **R-PYTEST-002** MUST: Test files MUST follow pytest naming conventions (test_*.py or *_test.py).
- **R-PYTEST-003** MUST: pytest MUST be listed as a dependency in project requirements files (requirements.txt, pyproject.toml, or setup.py).
- **R-PYTEST-004** SHOULD: Test files SHOULD import pytest or use pytest-specific features (fixtures, parametrize, marks).
- **R-PYTEST-005** SHOULD: CI/CD pipeline configuration SHOULD include pytest execution commands.
- **R-PYTEST-006** SHOULD: conftest.py files SHOULD be created at appropriate levels to define shared fixtures and configuration.
- **R-PYTEST-007** SHOULD: pytest.ini or pyproject.toml [tool.pytest.ini_options] SHOULD be configured to define test discovery paths, markers, and output options.
- **R-PYTEST-008** MAY: Alternative testing frameworks MAY be used for specialized scenarios (e.g., hypothesis for property-based testing, doctest for documentation tests) with documented justification.

### Verify

```bash
# Check for pytest imports in test files
grep -r "^import pytest" tests/ || grep -r "^from pytest" tests/

# Verify test file naming conventions
find tests/ -name 'test_*.py' -o -name '*_test.py' | head -5

# Collect and count tests
pytest --collect-only tests/ 2>&1 | grep -E '(test session starts|collected)'

# Verify pytest is in dependencies
grep -E "pytest|pytest-cov" requirements.txt pyproject.toml setup.py setup.cfg 2>/dev/null || echo 'Check dependency files'

# Verify tests/ directory structure mirrors source organization
find tests/ -type d | sort
```

**Accept when:**
- pytest is listed as a dependency in project requirements files (requirements.txt, pyproject.toml, or setup.py)
- Test files follow pytest naming conventions (test_*.py) and are organized in tests/ directory
- pytest --collect-only successfully discovers test files without errors
- CI/CD pipeline configuration includes pytest execution commands
- At least 80% of test files import pytest or use pytest-specific features (fixtures, parametrize, marks)
- tests/ directory structure mirrors source code organization
