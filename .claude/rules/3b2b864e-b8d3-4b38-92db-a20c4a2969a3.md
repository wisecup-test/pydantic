<rule_activation id="3b2b864e-b8d3-4b38-92db-a20c4a2969a3" title="Adopt Pydantic V1 Compatibility Layer for Type Validation: New Code Use" applies_to="**/*.py">
These rules are ALWAYS ACTIVE for all Python files. New code SHOULD use Pydantic V2 APIs unless V1 compatibility is explicitly required. All type validation, data serialization, API request/response validation, configuration parsing, and JSON schema generation MUST follow this decision.
</rule_activation>

### Rules

- **R-PYDANTIC-001** SHOULD: New code SHOULD use Pydantic V2 APIs (TypeAdapter, JSON schema generation) unless V1 compatibility is explicitly required.
- **R-PYDANTIC-002** MUST: All Pydantic V1 imports MUST use the `pydantic.v1` namespace, not direct `pydantic` imports (e.g., `from pydantic.v1 import BaseModel`).
- **R-PYDANTIC-003** MUST: Type checking tests MUST be placed in `tests/typechecking/` directory following existing patterns (type_adapter.py, json_schema_examples.py, pipeline_api.py).
- **R-PYDANTIC-004** MUST: Custom validators MUST be tested against pydantic-core validation engine.
- **R-PYDANTIC-005** SHOULD: Use `model_config` for Pydantic V2 configuration instead of Config class from V1.
- **R-PYDANTIC-006** SHOULD: Generate JSON schemas using `model.model_json_schema()` in V2 or `model.schema()` in V1 compatibility mode.
- **R-PYDANTIC-007** MUST: All data models requiring runtime validation MUST include validation logic.

### Verify

```bash
# Verify V1 imports are properly namespaced
grep -r 'from pydantic import' --include='*.py' . | grep -v 'from pydantic.v1 import' | grep -v 'from pydantic import TypeAdapter' | grep -v 'from pydantic import' || echo 'V1 imports properly namespaced'

# Verify type checking tests exist
find . -path '*/tests/typechecking/*.py' -type f | wc -l | awk '{if($1>0) print "Type checking tests found"; else exit 1}'

# Verify Pydantic imports work without conflicts
python -c 'import pydantic; from pydantic.v1 import BaseModel; from pydantic import TypeAdapter; print("Pydantic imports successful")'
```

**Accept when:**
- All Pydantic V1 imports use the `pydantic.v1` namespace, not direct pydantic imports
- Type checking test files exist in `tests/typechecking/` directory covering TypeAdapter, JSON schema, and pipeline APIs
- Pydantic and `pydantic.v1` modules can be imported successfully without conflicts
- Validator tests demonstrate compatibility with pydantic-core validation engine
- All data models include validation logic
- No direct `pydantic` imports found in legacy code paths (outside allowed contexts)
