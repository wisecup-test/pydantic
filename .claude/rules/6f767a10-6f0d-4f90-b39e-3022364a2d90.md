<rule_activation id="6f767a10-6f0d-4f90-b39e-3022364a2d90" title="Adopt Comprehensive Test Suite Organization for Core Library Validation: Tests Organized Support" applies_to="tests/**/*.py">
These rules are ALWAYS ACTIVE for all test files in the pydantic-core test suite. Tests MUST be organized hierarchically by functional domain (validators, serializers, arguments) with consistent naming conventions to support parallel CI/CD execution and maintainability.
</rule_activation>

### Rules

- **R-TSO-001** SHOULD: Tests SHOULD be organized to support parallel execution in CI/CD pipelines by maintaining test isolation and avoiding shared state.
- **R-TSO-002** SHOULD: Test files SHOULD follow the naming convention `test_<component>.py` and be organized in appropriate domain subdirectories (tests/validators/, tests/serializers/, tests/arguments_v3/).
- **R-TSO-003** SHOULD: Hierarchical test organization SHOULD be limited to 3 levels maximum (tests/<domain>/<subdomain>/test_<type>.py) to avoid excessive directory depth.
- **R-TSO-004** SHOULD: Cross-cutting tests that span multiple validators or domains SHOULD be placed in an integration test directory at the root level with clear documentation of their scope.
- **R-TSO-005** SHOULD: New validators SHOULD have corresponding test files following the established organizational pattern; tests SHOULD cover both unit-level validation and integration scenarios.
- **R-TSO-006** MAY: Legacy test files MAY deviate from naming conventions if they predate this ADR and refactoring would introduce regression risk (EXC-001).
- **R-TSO-007** MAY: Experimental or prototype validators MAY have co-located tests during initial development phase (EXC-002).

### Verify

```bash
# Count test files in standard organizational structure
find tests/ -type f -name 'test_*.py' | grep -E 'tests/(validators|serializers|arguments)/' | wc -l

# Collect all test modules and count
pytest tests/ --collect-only --quiet | grep -c '<Module'

# Run full test suite with verbose output and coverage
pytest tests/ -v --tb=short --maxfail=1

# Verify test file naming conventions
find tests/ -type f -name '*.py' | grep -v '__pycache__' | grep -v 'test_' | grep -v 'conftest.py' | grep -v '__init__.py'

# Check directory structure depth
find tests/ -type d | awk -F/ '{print NF}' | sort -n | tail -1
```

**Accept when:**
- All test files follow the naming convention `test_<component>.py` and are organized in appropriate domain subdirectories (tests/validators/, tests/serializers/, tests/arguments_v3/)
- Test discovery finds all test files and pytest can execute the full suite without import errors
- CI/CD pipeline successfully executes tests with clear pass/fail status and coverage reports
- Directory hierarchy does not exceed 3 levels (tests/<domain>/<subdomain>/test_<type>.py)
- New validator additions include corresponding test files in the appropriate domain directory
- Cross-cutting integration tests are documented and placed in a designated integration test directory

<enforcement>
Claude Code MUST verify test organization compliance before accepting changes to the test suite. Pull requests with incorrectly organized tests MUST receive automated guidance. New validators without corresponding tests MUST block merge until tests are added in the correct organizational structure.
</enforcement>
