<rule_activation id="9d4be7d4-c619-4269-a872-a6fc6537da64" title="Adopt Comprehensive Test Suite Organization for Core Library Validation: Core Library Tests" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files matching test suite organization patterns in pydantic-core and similar core validation libraries.
</rule_activation>

### Rules

- **R-CORE-001** MUST: Core library tests MUST be executable in the CI/CD pipeline with clear pass/fail criteria.
- **R-CORE-002** MUST: All test files within the pydantic-core/tests directory MUST follow the naming convention `test_<component>.py`.
- **R-CORE-003** MUST: Test files MUST be organized hierarchically by functional domain (validators, serializers, arguments) with specific test files for each validation type.
- **R-CORE-004** SHOULD: Tests SHOULD be organized in a maximum of 3 directory levels: `tests/<domain>/<subdomain>/test_<type>.py`.
- **R-CORE-005** SHOULD: Test functions SHOULD be prefixed with `test_` following pytest conventions.
- **R-CORE-006** SHOULD: Tests SHOULD be tagged with pytest markers (@pytest.mark) by category (unit, integration, slow) for selective CI/CD execution.
- **R-CORE-007** MAY: Legacy test files MAY deviate from naming conventions if they predate this ADR and refactoring would introduce regression risk (EXC-001).
- **R-CORE-008** MAY: Experimental or prototype validators MAY have co-located tests during initial development phase (EXC-002).

### Verify

```bash
# Count test files in standard domain directories
find tests/ -type f -name 'test_*.py' | grep -E 'tests/(validators|serializers|arguments)/' | wc -l

# Collect all test modules
pytest tests/ --collect-only --quiet | grep -c '<Module'

# Execute full test suite with verbose output
pytest tests/ -v --tb=short --maxfail=1
```

**Accept when:**
- All test files follow the naming convention `test_<component>.py` and are organized in appropriate domain subdirectories (validators, serializers, arguments)
- Test discovery finds all test files and pytest can execute the full suite without import errors
- CI/CD pipeline successfully executes tests with clear pass/fail status and coverage reports
- No validators lack corresponding test files (verified via coverage reports)
- Tests are organized in no more than 3 directory levels

<enforcement>
Claude Code MUST NOT skip or defer verification of test suite organization. All pull requests adding or modifying tests MUST pass the verify commands before acceptance. Violations receive automated comments with guidance; new validator additions without corresponding tests block merge until tests are added.
</enforcement>
