These rules are ALWAYS ACTIVE for all public/external API development and testing activities.

### Rules

- **R-API-001** MUST: JSON serialization and deserialization for public API data structures MUST be validated through unit tests.
- **R-API-002** MUST: All public API functions, methods, and classes exposed to external consumers require corresponding unit tests.
- **R-API-003** MUST: Validator functions and validation logic accessible through public interfaces must be tested.
- **R-API-004** MUST: Data serialization/deserialization methods (JSON, dict conversion) must be tested.
- **R-API-005** MUST: Error types and error handling mechanisms in public APIs must be tested.
- **R-API-006** MUST: Type hints and typing utilities exposed as public API surface must be tested.
- **R-API-007** SHOULD: Organize test files to mirror the structure of the API module being tested (e.g., validators/test_dict.py tests validators/dict.py).
- **R-API-008** SHOULD: Use descriptive test names that clearly indicate what contract or behavior is being validated.
- **R-API-009** SHOULD: Group related test cases using test classes or pytest markers to improve test organization and selective execution.
- **R-API-010** SHOULD: Include both positive test cases (valid inputs) and negative test cases (invalid inputs, edge cases, error conditions) for comprehensive coverage.
- **R-API-011** SHOULD: Leverage parametrized tests to efficiently cover multiple input variations without code duplication.
- **R-API-012** SHOULD: Document complex test scenarios with comments explaining the API contract being validated.

### Exceptions

- **EXC-001**: Deprecated API functions scheduled for removal in next major version
- **EXC-002**: Experimental APIs marked with explicit unstable/beta status

### Verify

```bash
# Run test suite with coverage analysis
pytest tests/ pydantic-core/tests/ -v --cov=. --cov-report=term-missing --cov-fail-under=90

# Count test files
find tests/ pydantic-core/tests/ -name 'test_*.py' | wc -l

# Count test functions
grep -r "def test_" tests/ pydantic-core/tests/ | wc -l
```

**Accept when:**
- All public API modules have corresponding test files with minimum 90% code coverage
- Test suite executes successfully with all tests passing in CI pipeline
- Each public API function/class has at least one positive test case and one negative/error test case
- Test files follow established naming conventions (test_<module>.py) and organizational structure
- Both positive test cases (valid inputs) and negative test cases (invalid inputs, edge cases, error conditions) are present

<enforcement>
Claude Code MUST NOT skip or defer verification. Code coverage analysis and test execution are mandatory before accepting changes to public API modules. Pull requests without adequate test coverage (minimum 90% for public API modules) MUST be blocked from merging.
</enforcement>
