<rule_activation id="a674356a-76aa-4464-b8c0-ff466bf5ba2d" title="Adopt Type Annotation and Runtime Validation for Public API Interfaces: Public Modules Pass" applies_to="**/*">
These rules are ALWAYS ACTIVE for all public and external API interfaces. All code that exposes public APIs must comply with the type annotation and validation requirements specified herein.
</rule_activation>

### Rules

- **R-API-001** MUST: All public API modules MUST pass mypy strict mode type checking without errors or suppressions.
- **R-API-002** MUST: All functions, methods, and classes exposed in public module `__all__` declarations require complete type annotations for parameters and return values.
- **R-API-003** MUST: All API endpoints, request/response models, and data transfer objects require type annotations and runtime validation.
- **R-API-004** MUST: All configuration classes and settings objects used by external consumers require type annotations.
- **R-API-005** MUST: All exception classes and error types that may be raised to API consumers require type annotations.
- **R-API-006** MUST: All callback interfaces and plugin extension points require type annotations.
- **R-API-007** SHOULD: Use Pydantic's BaseModel for data classes and validation, leveraging Field() for constraints and custom validators for complex business logic.
- **R-API-008** SHOULD: Create reusable type aliases and generic types for common patterns to maintain consistency across APIs.
- **R-API-009** MAY: Private module internals prefixed with underscore (_) that are not part of the public API are exempt from strict type checking requirements.
- **R-API-010** MAY: Test utilities and fixtures used exclusively within test suites are exempt from strict type checking requirements.

### Verify

```bash
# Verify all public API modules pass mypy strict mode
mypy --strict --config-file mypy.ini $(find . -name '*.py' -path '*/api/*' -o -path '*/public/*')

# Verify all public functions have return type annotations
grep -r 'def ' --include='*.py' api/ | grep -v '\->' | grep -v '^\s*#' || echo 'All public functions have return type annotations'

# Run type and validation tests
python -m pytest tests/test_typing.py tests/test_validation.py -v --tb=short
```

**Accept when:**
- All public API modules pass mypy strict mode checking with zero errors and zero `type: ignore` comments
- All public API functions have complete type annotations for parameters and return values verified by automated linting
- Runtime validation tests demonstrate that invalid inputs are rejected with clear error messages for all public API endpoints
- CI/CD pipeline runs mypy in strict mode on all public API modules and fails builds on type checking errors
- Pre-commit hooks validate type annotation completeness using automated linting tools
- Code review checklist includes verification of type annotations and runtime validation for all API changes
- Automated test suite includes type-focused tests that verify validation behavior and error handling

<enforcement>
Claude Code MUST NOT skip or defer verification of type annotations and runtime validation for public API interfaces. All public modules MUST pass mypy strict mode checking. Violations block pull request merges and require architectural review board approval for exceptions.
