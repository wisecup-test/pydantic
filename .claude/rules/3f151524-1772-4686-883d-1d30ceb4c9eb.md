<rule_activation id="3f151524-1772-4686-883d-1d30ceb4c9eb" title="Adopt Comprehensive Test Suite Organization for Core Library Validation: Test Files Organized" applies_to="tests/**/*.py">
These rules are ALWAYS ACTIVE for all test files within the pydantic-core/tests directory. Test files MUST be organized hierarchically by functional domain (validators, serializers, arguments) with dedicated subdirectories for each category.
</rule_activation>

### Rules

- **R-TSO-001** MUST: Test files MUST be organized hierarchically by functional domain (validators, serializers, arguments) with dedicated subdirectories for each category.
- **R-TSO-002** MUST: Test files MUST follow the naming convention `test_<component>.py` and be placed in appropriate domain subdirectories.
- **R-TSO-003** MUST: All test files within the pydantic-core/tests directory MUST adhere to this organizational structure.
- **R-TSO-004** SHOULD: Use pytest's test discovery conventions with test function names prefixed with `test_`.
- **R-TSO-005** SHOULD: Implement pytest markers (@pytest.mark) to tag tests by category (unit, integration, slow) for selective CI/CD execution.
- **R-TSO-006** MAY: Legacy test files may deviate from naming conventions if they predate this ADR and refactoring would introduce regression risk (EXC-001).
- **R-TSO-007** MAY: Experimental or prototype validators may have co-located tests during initial development phase (EXC-002).

### Verify

```bash
# Count test files in standard domain directories
find tests/ -type f -name 'test_*.py' | grep -E 'tests/(validators|serializers|arguments)/' | wc -l

# Collect all test modules
pytest tests/ --collect-only --quiet | grep -c '<Module'

# Execute full test suite with verbose output
pytest tests/ -v --tb=short --maxfail=1
```

**Accept when:**
- All test files follow the naming convention `test_<component>.py` and are organized in appropriate domain subdirectories (validators, serializers, arguments).
- Test discovery finds all test files and pytest can execute the full suite without import errors.
- CI/CD pipeline successfully executes tests with clear pass/fail status and coverage reports.
- Directory structure does not exceed 3 levels maximum (tests/<domain>/<subdomain>/test_<type>.py).
- Coverage reports identify validators without corresponding test files.
