<rule_activation id="8c78a1d5-25c1-4358-af91-3808f9a72a8e" title="Adopt pytest as Standard Testing Framework for Python Projects: Projects Use Pytest" applies_to="**/*">
These rules are ALWAYS ACTIVE for all Python projects. pytest is the standard testing framework for all Python test files in the tests/ directory and CI/CD pipelines.
</rule_activation>

### Rules

- **R-PYTEST-001** MAY: Projects MAY use pytest plugins (e.g., pytest-cov, pytest-xdist) to extend testing capabilities
- **R-PYTEST-002** MUST: All Python test files in the tests/ directory MUST follow pytest naming conventions (test_*.py or *_test.py)
- **R-PYTEST-003** MUST: Test files MUST be organized under a dedicated tests/ directory with subdirectories for functional areas (validators, serializers, etc.)
- **R-PYTEST-004** SHOULD: Projects SHOULD use pytest fixtures for dependency injection and test isolation
- **R-PYTEST-005** SHOULD: Projects SHOULD use pytest parametrization for testing multiple input scenarios
- **R-PYTEST-006** MUST: pytest MUST be listed as a dependency in project requirements files (requirements.txt, pyproject.toml, or setup.py)
- **R-PYTEST-007** SHOULD: Projects SHOULD configure pytest via pytest.ini or pyproject.toml [tool.pytest.ini_options] section
- **R-PYTEST-008** SHOULD: Projects SHOULD create conftest.py files at appropriate levels to define shared fixtures and configuration
- **R-PYTEST-009** MUST: CI/CD pipelines MUST include pytest execution commands for test validation
- **R-PYTEST-010** SHOULD: Projects SHOULD use pytest-cov plugin for code coverage reporting and enforce minimum coverage thresholds

### Verify

```bash
# Check for pytest imports in test files
grep -r "^import pytest" tests/ || grep -r "^from pytest" tests/

# Verify test file naming conventions
find tests/ -name 'test_*.py' -o -name '*_test.py' | head -5

# Collect and verify pytest can discover tests
pytest --collect-only tests/ 2>&1 | grep -E '(test session starts|collected)'

# Verify pytest is in dependencies
grep -E "pytest|pytest-cov" requirements.txt pyproject.toml setup.py setup.cfg 2>/dev/null || echo 'Check dependency files'
```

**Accept when:**
- pytest is listed as a dependency in project requirements files (requirements.txt, pyproject.toml, or setup.py)
- Test files follow pytest naming conventions (test_*.py) and are organized in tests/ directory
- pytest --collect-only successfully discovers test files without errors
- CI/CD pipeline configuration includes pytest execution commands
- At least 80% of test files import pytest or use pytest-specific features (fixtures, parametrize, marks)

Claude Code MUST verify pytest compliance before approving Python test file changes. Verification is mandatory and MUST NOT be skipped or deferred. All new test files MUST follow pytest conventions and be discoverable by pytest --collect-only.
