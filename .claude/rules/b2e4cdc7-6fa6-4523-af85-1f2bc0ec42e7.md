<rule_activation id="b2e4cdc7-6fa6-4523-af85-1f2bc0ec42e7" title="Standardize Pytest-Based Validator Testing in CI/CD Pipeline: Test Files Include" applies_to="**/*">
These rules are ALWAYS ACTIVE for all validator test files and CI/CD pipeline configurations in pydantic-core.
</rule_activation>

### Rules

- **R-PYTEST-001** MAY: Test files MAY include parametrized tests to efficiently cover multiple input variations

### Verify

```bash
# Verify pytest test files are discoverable in validators directory
find tests/validators -name 'test_*.py' | wc -l | grep -v '^0$'

# Verify pytest can collect tests from validators directory
pytest tests/validators/ --collect-only --quiet | grep -c 'test session starts'

# Verify pytest is imported in validator test files
grep -r 'import pytest' tests/validators/ | wc -l | grep -v '^0$'
```

**Accept when:**
- All validator test files are discoverable by pytest in tests/validators/ directory
- CI/CD pipeline successfully executes pytest and reports test results
- Test files contain boundary testing for external client inputs as evidenced by test function names or assertions
- Test files include both positive test cases (valid inputs) and negative test cases (invalid inputs with expected errors)
- Pytest fixtures are used to share common test setup across validator test files

<enforcement>
Claude Code MUST NOT skip or defer verification. All validator implementations MUST have corresponding pytest-based test files in tests/validators/ directory. Pull requests without validator tests MUST be blocked from merging by CI/CD checks. Coverage drops below threshold MUST trigger automated notifications to PR authors.
</enforcement>
