### Rules

- **R-API-001** MAY: Tests MAY include structural pattern matching validation when APIs support pattern matching interfaces.

### Verify

```bash
# Verify test coverage meets minimum threshold
pytest tests/ pydantic-core/tests/ -v --cov=. --cov-report=term-missing --cov-fail-under=90

# Count test files
find tests/ pydantic-core/tests/ -name 'test_*.py' | wc -l

# Count test functions
grep -r "def test_" tests/ pydantic-core/tests/ | wc -l
```

**Accept when:**
- All public API modules have corresponding test files with minimum 90% code coverage
- Test suite executes successfully with all tests passing in CI pipeline
- Each public API function/class has at least one positive test case and one negative/error test case
- Test files follow established naming conventions (test_<module>.py) and organizational structure
- Tests cover validators, error handling, JSON serialization, and structural pattern matching for API surface areas

Claude Code MUST NOT skip or defer verification. Code coverage analysis in CI pipeline is mandatory with minimum 90% threshold enforcement for public API modules. Pull requests without adequate test coverage MUST be blocked from merging.
