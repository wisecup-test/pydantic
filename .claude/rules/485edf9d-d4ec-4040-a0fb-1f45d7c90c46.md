<rule_activation id="485edf9d-d4ec-4040-a0fb-1f45d7c90c46" title="Standardize Pytest-Based Validator Testing in CI/CD Pipeline: Validator Tests Cover" applies_to="**/*">
These rules are ALWAYS ACTIVE for all validator test files and CI/CD pipeline configurations in pydantic-core.
</rule_activation>

### Rules

- **R-VAL-001** SHOULD: Validator tests SHOULD cover both valid and invalid input scenarios to ensure comprehensive validation coverage

### Verify

```bash
# Verify validator test files are discoverable
find tests/validators -name 'test_*.py' | wc -l | grep -v '^0$'

# Verify pytest can collect tests
pytest tests/validators/ --collect-only --quiet | grep -c 'test session starts'

# Verify pytest is imported in test files
grep -r 'import pytest' tests/validators/ | wc -l | grep -v '^0$'
```

**Accept when:**
- All validator test files are discoverable by pytest in tests/validators/ directory
- CI/CD pipeline successfully executes pytest and reports test results
- Test files contain boundary testing for external client inputs as evidenced by test function names or assertions
- Both positive test cases (valid inputs) and negative test cases (invalid inputs with expected errors) are present
- Test coverage reports show adequate coverage for validator modules

<enforcement>
Claude Code MUST NOT skip or defer verification. Validator tests without both valid and invalid input scenarios MUST be flagged for remediation. CI/CD pipeline pytest execution is mandatory before merge.
</enforcement>
