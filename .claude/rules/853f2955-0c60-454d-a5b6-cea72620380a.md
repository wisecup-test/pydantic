<rule_activation id="853f2955-0c60-454d-a5b6-cea72620380a" title="Adopt Comprehensive Test Suite Organization for Core Library Validation: Test Suites Include" applies_to="tests/**/*.py">
These rules are ALWAYS ACTIVE for all test files within the pydantic-core/tests directory and establish a comprehensive, hierarchical test organization pattern for core library validation.
</rule_activation>

### Rules

- **R-TSO-001** MAY: Test suites MAY include integration tests that validate cross-cutting concerns like typing, docstrings, and build processes.
- **R-TSO-002** MUST: Test files MUST follow the naming convention `test_<component>.py` and be organized in appropriate domain subdirectories (validators, serializers, arguments).
- **R-TSO-003** MUST: All test files within the pydantic-core/tests directory MUST be organized hierarchically by functional domain (validators, serializers, arguments) with specific test files for each validation type.
- **R-TSO-004** SHOULD: Test suites SHOULD include both unit tests for individual validators and integration tests for complex scenarios like lax/strict mode handling and model root validation.
- **R-TSO-005** SHOULD: Directory hierarchy SHOULD be limited to 3 levels maximum (tests/<domain>/<subdomain>/test_<type>.py) to avoid excessive navigation complexity.
- **R-TSO-006** SHOULD: Integration tests for cross-cutting concerns SHOULD be placed in a dedicated integration test directory at the root level rather than scattered across domain subdirectories.
- **R-TSO-007** MUST: New validator additions MUST have corresponding test files; validators without tests block merge until tests are added.
- **R-TSO-008** SHOULD: Pytest markers (@pytest.mark) SHOULD be used to tag tests by category (unit, integration, slow) for selective CI/CD execution.

### Verify

```bash
# Count test files in standard domain directories
find tests/ -type f -name 'test_*.py' | grep -E 'tests/(validators|serializers|arguments)/' | wc -l

# Collect all test modules and count them
pytest tests/ --collect-only --quiet | grep -c '<Module'

# Run full test suite with verbose output and short traceback
pytest tests/ -v --tb=short --maxfail=1

# Verify test file naming conventions
find tests/ -type f -name '*.py' | grep -v 'test_' | grep -v '__pycache__' | grep -v 'conftest.py'

# Check for tests in non-standard locations
find tests/ -type f -name 'test_*.py' | grep -v -E 'tests/(validators|serializers|arguments|integration)/'
```

**Accept when:**
- All test files follow the naming convention `test_<component>.py` and are organized in appropriate domain subdirectories (validators, serializers, arguments, or integration)
- Test discovery finds all test files and pytest can execute the full suite without import errors
- CI/CD pipeline successfully executes tests with clear pass/fail status and coverage reports
- No test files exist in non-standard locations outside the defined domain structure
- Coverage reports show clear mapping between validators and their corresponding test files
- New validators have corresponding test files before merge
