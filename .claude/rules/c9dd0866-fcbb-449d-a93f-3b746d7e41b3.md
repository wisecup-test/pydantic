These rules are ALWAYS ACTIVE for all public API contracts, external integrations, and type validation implementations. All code that exposes public APIs or validates external data MUST comply with these rules.

### Rules

- **R-PSV-001** MUST: Public API contracts MUST be validated with mypy in strict mode or equivalent static type checkers during CI/CD pipeline execution.
- **R-PSV-002** MUST: All REST API endpoints exposed to external consumers require type validation.
- **R-PSV-003** MUST: GraphQL schemas and resolvers accessible outside the organization require type validation.
- **R-PSV-004** MUST: SDK and library public interfaces require type validation.
- **R-PSV-005** MUST: Webhook handlers receiving data from third-party services require type validation.
- **R-PSV-006** MUST: Data ingestion pipelines processing external data sources require type validation.
- **R-PSV-007** MUST: Configuration files and environment variables affecting public API behavior require type validation.
- **R-PSV-008** SHOULD: Use Pydantic V2 for new implementations to benefit from performance improvements via Rust core (pydantic-core).
- **R-PSV-009** SHOULD: Create shared base models and validators for common patterns (pagination, timestamps, IDs) to ensure consistency across APIs.
- **R-PSV-010** SHOULD: Implement comprehensive error handling that translates validation errors into appropriate HTTP status codes (400 Bad Request) with detailed error messages.
- **R-PSV-011** SHOULD: Document all public models with docstrings and examples that are automatically included in API documentation.

### Verify

```bash
# Count Pydantic models in codebase
grep -r "class.*BaseModel" --include="*.py" | wc -l

# Verify static type checking with mypy strict mode
mypy --strict --config-file mypy.ini . 2>&1 | grep -E "(error|Success)"

# Run type validation tests
pytest tests/test_typing.py tests/test_errors.py -v

# Count custom validators
grep -r "@validator\|@field_validator" --include="*.py" | wc -l
```

**Accept when:**
- All public API endpoints have corresponding Pydantic models with 100% field coverage
- Mypy strict mode passes with zero errors for all public API modules
- Test coverage for validation logic exceeds 90% with tests for invalid inputs, edge cases, and error messages
- CI/CD pipeline includes automated validation checks that fail builds on type errors or missing validation
