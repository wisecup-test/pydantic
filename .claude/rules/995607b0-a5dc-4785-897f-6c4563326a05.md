<rule_activation id="995607b0-a5dc-4785-897f-6c4563326a05" title="Adopt Environment Variable-Based Secret Management for External API Integration: Code That Accesses" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files matching `**/*`. Code that accesses external APIs must handle credentials securely using environment variables, never hardcoding secrets in source code.
</rule_activation>

### Rules

- **R-SEC-001** MUST: Code that accesses external APIs MUST handle missing or invalid credentials gracefully with clear error messages.
- **R-SEC-002** MUST: All external API integrations requiring authentication MUST retrieve credentials from environment variables at runtime, never from hardcoded values in source code.
- **R-SEC-003** MUST: API keys, tokens, OAuth credentials, and other authentication secrets MUST NOT be committed to version control.
- **R-SEC-004** SHOULD: Use consistent naming conventions for environment variables: SERVICE_NAME_API_KEY, SERVICE_NAME_TOKEN, SERVICE_NAME_SECRET.
- **R-SEC-005** SHOULD: Create a .env.example file documenting all required environment variables with placeholder values.
- **R-SEC-006** SHOULD: Implement startup validation that checks for required environment variables and fails fast with clear error messages.
- **R-SEC-007** MAY: Unit tests may use hardcoded mock credentials when testing against mock API servers with no external connectivity (EXC-001).

### Verify

```bash
# Check for hardcoded API credentials in Python files
grep -r "api[_-]key\s*=\s*['\"]" --include="*.py" --exclude-dir=tests --exclude-dir=.venv | grep -v "os.environ" | grep -v "getenv"

# Check for hardcoded tokens in Python files
grep -r "token\s*=\s*['\"]" --include="*.py" --exclude-dir=tests --exclude-dir=.venv | grep -v "os.environ" | grep -v "getenv"

# Verify .env files are properly excluded from version control
find . -name ".env" -not -path "*/node_modules/*" -not -path "*/.venv/*" | xargs git check-ignore -v
```

**Accept when:**
- No hardcoded API credentials are found in source code (excluding test mocks)
- All .env files containing actual secrets are properly excluded from version control
- All external API integrations retrieve credentials from environment variables at runtime
- Documentation clearly specifies required environment variables for each integration
- Error messages gracefully handle missing or invalid credentials without exposing sensitive information

<enforcement>
Claude Code MUST NOT skip or defer verification. All pull requests containing hardcoded credentials must be rejected. Exposed credentials must be revoked and regenerated before code can be merged. Security team must be notified of any credential exposure incidents.
</enforcement>
