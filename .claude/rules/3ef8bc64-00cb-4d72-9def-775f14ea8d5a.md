<rule_activation id="3ef8bc64-00cb-4d72-9def-775f14ea8d5a" title="Standardize Pytest-Based Validator Testing in CI/CD Pipeline: Test Files Follow" applies_to="tests/**/*.py">
These rules are ALWAYS ACTIVE for all validator test files in the pydantic-core project. Test files MUST follow pytest-based patterns with external client boundary testing to ensure consistent validation behavior across the CI/CD pipeline.
</rule_activation>

### Rules

- **R-PYTEST-001** SHOULD: Test files SHOULD follow the naming convention `test_<validator_name>.py` for consistency and discoverability.
- **R-PYTEST-002** MUST: All validator test files MUST be located in `tests/` or `tests/validators/` directories.
- **R-PYTEST-003** MUST: Test files MUST include both positive test cases (valid inputs) and negative test cases (invalid inputs with expected errors) for comprehensive coverage.
- **R-PYTEST-004** SHOULD: Test files SHOULD use pytest fixtures to share common test setup across validator test files, reducing duplication.
- **R-PYTEST-005** MUST: Tests MUST validate behavior at external client boundaries where external inputs are processed.
- **R-PYTEST-006** SHOULD: Tests SHOULD be organized by validator type in the `tests/validators/` directory structure for clear navigation.
- **R-PYTEST-007** MUST: CI/CD pipeline MUST execute pytest and report test results before merge.
- **R-PYTEST-008** MUST: Test coverage MUST not drop below established thresholds; violations trigger automated notifications.

### Verify

```bash
# Verify test file naming convention and discoverability
find tests/validators -name 'test_*.py' | wc -l | grep -v '^0$'

# Verify pytest can collect all tests
pytest tests/validators/ --collect-only --quiet | grep -c 'test session starts'

# Verify pytest is imported in test files
grep -r 'import pytest' tests/validators/ | wc -l | grep -v '^0$'
```

**Accept when:**
- All validator test files are discoverable by pytest in `tests/validators/` directory
- CI/CD pipeline successfully executes pytest and reports test results
- Test files contain boundary testing for external client inputs as evidenced by test function names or assertions
- Test files follow the `test_<validator_name>.py` naming convention
- Both positive and negative test cases are present for each validator
