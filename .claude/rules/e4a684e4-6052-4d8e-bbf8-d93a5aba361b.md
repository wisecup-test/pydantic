<rule_activation id="e4a684e4-6052-4d8e-bbf8-d93a5aba361b" title="Adopt Comprehensive Test Suite Organization for Core Library Validation: Each Validator Type" applies_to="tests/**/*.py">
These rules are ALWAYS ACTIVE for all test files within the pydantic-core/tests directory. Test organization must follow a hierarchical, domain-based structure with dedicated test files for each validator type.
</rule_activation>

### Rules

- **R-TSO-001** MUST: Each validator type MUST have a dedicated test file following the naming convention `test_<validator_type>.py` (e.g., `test_float.py`, `test_complex.py`, `test_frozenset.py`)
- **R-TSO-002** MUST: All test files within the pydantic-core/tests directory MUST be organized hierarchically by functional domain (validators, serializers, arguments) with specific test files for each validation type
- **R-TSO-003** MUST: Test files MUST use pytest naming conventions: prefix test files with `test_` and test functions with `test_`
- **R-TSO-004** SHOULD: Implement pytest markers (@pytest.mark) to tag tests by category (unit, integration, slow) for selective CI/CD execution
- **R-TSO-005** SHOULD: Limit directory hierarchy to 3 levels maximum (tests/<domain>/<subdomain>/test_<type>.py) to avoid excessive navigation complexity
- **R-TSO-006** MAY: Allow integration test directory at root level for cross-cutting concerns that span multiple validators or domains

### Verify

```bash
# Count test files in standard locations
find tests/ -type f -name 'test_*.py' | grep -E 'tests/(validators|serializers|arguments)/' | wc -l

# Collect all test modules
pytest tests/ --collect-only --quiet | grep -c '<Module'

# Execute full test suite with verbose output
pytest tests/ -v --tb=short --maxfail=1
```

**Accept when:**
- All test files follow the naming convention `test_<component>.py` and are organized in appropriate domain subdirectories (validators/, serializers/, arguments/)
- Test discovery finds all test files and pytest can execute the full suite without import errors
- CI/CD pipeline successfully executes tests with clear pass/fail status and coverage reports
- Coverage reports show no validators without corresponding test files
- Directory structure does not exceed 3 levels of nesting
