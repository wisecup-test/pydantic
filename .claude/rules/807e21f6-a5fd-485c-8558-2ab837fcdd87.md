<rule_activation id="807e21f6-a5fd-485c-8558-2ab837fcdd87" title="Standardize Pytest-Based Validator Testing in CI/CD Pipeline: Validator Implementations Have" applies_to="**/*">
These rules are ALWAYS ACTIVE for all validator implementations and test files in the pydantic-core project.
</rule_activation>

### Rules

- **R-VAL-001** MUST: All validator implementations MUST have corresponding pytest-based test files in the tests/validators/ directory
- **R-VAL-002** MUST: Test files MUST include both positive test cases (valid inputs) and negative test cases (invalid inputs with expected errors) for comprehensive coverage
- **R-VAL-003** MUST: Tests MUST validate behavior at external client boundaries where external inputs are processed
- **R-VAL-004** SHOULD: Use pytest fixtures to share common test setup across validator test files, reducing duplication
- **R-VAL-005** SHOULD: Organize tests by validator type in the tests/validators/ directory structure for clear navigation
- **R-VAL-006** SHOULD: Configure pytest in CI/CD to fail fast on first error during development, but run all tests in pre-merge validation
- **R-VAL-007** SHOULD: Document expected test patterns in CONTRIBUTING.md to guide new contributors

### Verify

```bash
# Verify pytest test files exist for validators
find tests/validators -name 'test_*.py' | wc -l | grep -v '^0$'

# Verify pytest can discover tests
pytest tests/validators/ --collect-only --quiet | grep -c 'test session starts'

# Verify pytest is imported in test files
grep -r 'import pytest' tests/validators/ | wc -l | grep -v '^0$'
```

**Accept when:**
- All validator test files are discoverable by pytest in tests/validators/ directory
- CI/CD pipeline successfully executes pytest and reports test results
- Test files contain boundary testing for external client inputs as evidenced by test function names or assertions
- New validators include corresponding test files before merge
- Test coverage meets or exceeds project threshold

Claude Code MUST NOT skip or defer verification. All validator implementations MUST have corresponding pytest-based test files. Pull requests without validator tests are blocked from merging by CI/CD checks. Violations require written justification and technical lead approval.
